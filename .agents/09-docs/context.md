# Documentation - Context

## Docs Structure

docs/
├── README.md                  ← Project overview and quick start
├── architecture.md            ← System design and service boundaries
├── api-reference.md           ← All HTTP endpoints
├── events.md                  ← All async event contracts
└── local-setup.md             ← How to run locally with Aspire

## Audience
- New developers onboarding to the team
- Developers integrating with the APIs
- Reviewers evaluating the architecture

## Conventions
- Use Mermaid diagrams for architecture visuals
- Use tables for API and event contracts
- Use code blocks for request/response examples
- Keep each doc focused on one concern
